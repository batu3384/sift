# Code of Conduct

## Our standard

This project expects direct, respectful, and technically honest collaboration.
Contributors should:

- discuss ideas and code without personal attacks
- keep reviews factual and specific
- assume good intent, but challenge weak reasoning clearly
- respect platform, accessibility, and safety constraints

## Unacceptable behavior

The following are not acceptable in this project space:

- harassment, intimidation, or discrimination
- insulting, demeaning, or hostile language
- publishing private information without permission
- derailing technical discussion into personal conflict
- deliberately unsafe or destructive guidance presented as acceptable

## Maintainer responsibilities

Project maintainers may edit, remove, or reject comments, issues, pull requests,
or other contributions that violate this code of conduct. They may also limit or
end participation for repeated or severe violations.

## Reporting

To report a conduct issue, use the contact path in [SECURITY.md](SECURITY.md) or
open a private maintainer contact through GitHub if available.
